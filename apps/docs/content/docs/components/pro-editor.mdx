## Usage

```tsx
import { useState } from "react"

import { ProEditor } from "@/components/pro/pro-editor"

export default function EditorPage() {
  const [value, setValue] = useState("")

  return (
    <ProEditor
      language="tsx"
      theme="dark"
      value={value}
      onChange={setValue}
      height={400}
    />
  )
}
```

`ProEditor` receives `language` and `theme` as controlled props. Keep language and theme switchers in your page or demo state and pass the selected values into `ProEditor`; the built-in toolbar displays the current language but does not own those controls.

`ProEditor` does not ship a built-in preview renderer. Pass `preview.component` from the outside when you want preview or split mode; omit `preview` for edit-only usage.

Omit `height` when the editor should fill its parent container; set an explicit parent height with CSS such as `h-[520px]`, `h-full`, or `flex-1`.

The toolbar is part of `ProEditor`. Use `toolbar.actions` for custom actions and `toolbar.options` to hide built-in controls. Action buttons default to `type="button"`; pass `type: "submit"` only when the editor lives inside a form and the action should submit it. Set `toolbar.options` to `false` to hide all built-in toolbar buttons.

## Examples

### Markdown

```tsx
import { MarkdownViewer } from "@/components/pro/pro-viewer/markdown"

<ProEditor
  language="markdown"
  theme="dark"
  value={markdown}
  onChange={setMarkdown}
  preview={{
    component: ({ content }) => <MarkdownViewer content={content} theme="dark" className="p-4" />,
  }}
  height={400}
/>
```

### Without Preview

```tsx
<ProEditor language="typescript" theme="dark" value={code} onChange={setCode} />
```

### Controlled View Mode

```tsx
const [mode, setMode] = useState<"edit" | "preview" | "split">("split")
const [theme, setTheme] = useState<"light" | "dark">("dark")

<ProEditor
  language="markdown"
  theme={theme}
  value={code}
  onChange={setCode}
  preview={{
    component: ({ content }) => <MarkdownViewer content={content} theme={theme} className="p-4" />,
    mode,
    onModeChange: setMode,
  }}
/>
```

### Toolbar Extension

```tsx
<ProEditor
  language="typescript"
  value={code}
  onChange={setCode}
  toolbar={{
    actions: [
      {
        key: "run",
        label: "Run",
        onClick: ({ value }) => runCode(value),
      },
      {
        key: "prettier",
        label: "Prettier",
        position: "after",
        onClick: ({ format }) => format(),
      },
    ],
  }}
/>
```

## API Reference

### ProEditor

| Prop          | Type                                  | Default        | Description                                            |
| ------------- | ------------------------------------- | -------------- | ------------------------------------------------------ |
| `value`       | `string`                              | `""`           | Controlled editor content                              |
| `onChange`    | `(value: string) => void`             | —              | Content change callback                                |
| `language`    | `string`                              | Required       | Language id; use `tsx` for React/JSX snippets and `typescript` for plain TypeScript |
| `theme`       | `'light' \| 'dark'`          | `'dark'`         | Editor theme. Internally maps to One Light / One Dark Pro syntax themes. |
| `className`   | `string`                              | —              | Additional CSS class for the editor root. |
| `height`      | `string \| number`                    | —              | Content height. When omitted, the editor fills its parent container. |
| `toolbar`     | `false \| { actions?, options? }`     | —              | Toolbar config. Pass `false` to hide the toolbar; pass `options: false` to hide built-in buttons. |
| `preview`     | `{ component, mode?, defaultMode?, onModeChange? }` | — | External preview renderer and preview mode state; enables preview and split modes. |
