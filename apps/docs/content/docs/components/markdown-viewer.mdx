## Usage

```tsx
import { MarkdownViewer } from "@/components/pro/pro-viewer/markdown"

export default function Example() {
  return (
    <MarkdownViewer
      content="# Hello\n\n> [!NOTE]\n> GitHub-style markdown is supported."
    />
  )
}
```

## API Reference

### MarkdownViewer

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `content` | `string` | — | Markdown string to render. |
| `theme` | `"light" \| "dark"` | `"dark"` | Code block surface theme. Pass this from your app theme when needed. |
| `emptyText` | `string` | `"No markdown content"` | Message shown when `content` is empty. |
| `className` | `string` | — | Additional CSS class for the prose wrapper. |

## Supported Syntax

- CommonMark plus GitHub Flavored Markdown: tables, task lists, strikethrough, autolinks, and footnotes.
- GitHub alerts: `NOTE`, `TIP`, `IMPORTANT`, `WARNING`, and `CAUTION`.
- Table of contents generation from `## Contents`, `## Table of Contents`, or `## TOC` headings.
- Raw HTML rendered through a sanitize pipeline.
- Inline and block math rendered with MathJax.
- Fenced code blocks rendered through CodeViewer, including shared Shiki themes, copy controls, line numbers, folding, and empty/error states.
