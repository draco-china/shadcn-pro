## Usage

Every field is a plain React/shadcn component. Simple fields preserve the underlying shadcn/ui prop surface where practical; shadcn-pro adds only small behaviors such as `allowClear`, prefix/suffix slots, options mapping, and composed field layouts. Formily adapters live under `pro-form/formily-fields`, so schema-driven forms can use the same field primitives without making the base fields depend on Formily.

```tsx
import { Input } from "@/components/pro/pro-fields/input"

export default function Example() {
  return <Input placeholder="Enter username" allowClear />
}
```

For schema-driven forms, install `pro-form` and import the Formily adapters from `pro-form/formily-fields`.

```tsx
import { createForm } from "@formily/core"
import { Field, FormProvider } from "@formily/react"

import { FormItem } from "@/components/pro/pro-form/form-item"
import { FormilyInput } from "@/components/pro/pro-form/formily-fields"

const form = createForm()

export default function Example() {
  return (
    <FormProvider form={form}>
      <Field
        name="username"
        title="Username"
        decorator={[FormItem]}
        component={[FormilyInput, { placeholder: "Enter username" }]}
      />
    </FormProvider>
  )
}
```

## FacetedFilter

A Popover + Checkbox filter component designed for faceted search — auto-used by `ProTable` when `meta.filters` is set on a column, and available standalone for forms or custom toolbars. Supports single-select and multi-select modes.

### API Reference

| Prop          | Type                                | Default     | Description                                              |
| ------------- | ----------------------------------- | ----------- | -------------------------------------------------------- |
| `value`       | `string \| string[]`                | —           | Controlled selected value(s)                             |
| `onChange`    | `(value: string \| string[] \| undefined) => void` | — | Change handler                              |
| `options`     | `{ label: string; value: string; icon?: ComponentType }[]` | `[]` | Filter options                      |
| `placeholder` | `string`                            | `'Filter...'` | Button label / search placeholder                      |
| `mode`        | `'single' \| 'multi'`               | `'multi'`   | Single-select (radio-like) or multi-select (checkbox)    |
| `className`   | `string`                            | —           | Additional CSS classes                                   |

```tsx
import { FacetedFilter } from "@/components/pro/pro-fields/faceted-filter"

<FacetedFilter
  placeholder="Status"
  mode="multi"
  options={[
    { label: "Active", value: "active" },
    { label: "Inactive", value: "inactive" },
  ]}
  value={selected}
  onChange={setSelected}
/>
```

### ArrayField

Drag-to-reorder list with built-in state management. Supports controlled (`value` + `onChange`) and uncontrolled (`defaultValue`) usage.

<ComponentPreview
  name="pro-fields-array-field-demo"
  previewClassName="min-h-[280px] items-start p-6"
  align="start"
/>

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `TItem[]` | — | Controlled array value |
| `onChange` | `(value: TItem[]) => void` | — | Called on any change |
| `defaultValue` | `TItem[]` | `[]` | Initial value (uncontrolled) |
| `newItem` | `() => TItem` | `() => ''` | Factory for blank items |
| `renderItem` | `(item, index, helpers) => ReactNode` | — | Render each item; helpers: `update`, `remove` |
| `addText` | `ReactNode` | `'Add item'` | Add button label |
| `max` | `number` | — | Hide Add button when reached |
| `min` | `number` | `0` | Hide Remove button when at minimum |
| `disabled` | `boolean` | `false` | Disables all controls |

### ObjectField

|------|------|---------|-------------|
