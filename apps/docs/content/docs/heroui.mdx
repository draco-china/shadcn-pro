## Overview

shadcn-pro supports **HeroUI** as a drop-in replacement for shadcn/ui primitives.
When you run `add` with `--base heroui`, the CLI downloads the same component source
and rewrites every `@/components/ui/<name>` import to the corresponding `@heroui/<name>` package.

No separate component registry needed — one source, two bases.

## Setup

### 1. Install HeroUI

```bash
bun add @heroui/react framer-motion
```

Follow the [HeroUI installation guide](https://heroui.com/docs/guide/installation) to configure
your `tailwind.config.ts` and wrap your app with `<HeroUIProvider>`.

### 2. Initialize shadcn-pro with HeroUI

```bash
npx @draco-china/shadcn-pro init
```

When prompted, select **HeroUI** as your base UI library.
This writes `"ui": "heroui"` and the full alias map to `shadcn-pro.json`.

### 3. Add components

```bash
npx @draco-china/shadcn-pro add image-viewer
# or override per-command:
npx @draco-china/shadcn-pro add image-viewer --base heroui
```

The CLI automatically rewrites imports, e.g.:

```diff
- import { Button } from '@/components/ui/button'
+ import { Button } from '@heroui/button'
```

## Alias Map

The built-in alias map covers the most common shadcn/ui primitives:

| shadcn/ui | HeroUI |
|---|---|
| `button` | `@heroui/button` |
| `slider` | `@heroui/slider` |
| `dropdown-menu` | `@heroui/dropdown` |
| `separator` | `@heroui/divider` |
| `badge` | `@heroui/badge` |
| `tooltip` | `@heroui/tooltip` |
| `tabs` | `@heroui/tabs` |
| `input` | `@heroui/input` |
| `card` | `@heroui/card` |
| `modal` | `@heroui/modal` |
| `progress` | `@heroui/progress` |

You can extend or override the map by editing the `aliases` field in `shadcn-pro.json`.

## Custom Base

You can point `--base` at any custom UI library by configuring `aliases` manually:

```json
{
  "ui": "my-ui",
  "aliases": {
    "button": "@my-ui/button",
    "input": "@my-ui/input"
  }
}
```

```bash
npx @draco-china/shadcn-pro add pro-form --base my-ui
```
